# YouTubeDownloader 后端接口文档

## 1. 基础信息
- 协议：`HTTP` + `WebSocket`
- 默认地址：`http://127.0.0.1:8000`
- 数据格式：`application/json`
- 认证方式：当前版本无鉴权（仅本地桌面应用调用）

## 2. 通用说明
- 时间字段使用 ISO 格式字符串，例如：`2026-05-11T10:20:30.123456`
- 常见错误结构：

```json
{
  "detail": "错误描述"
}
```

- 常见状态码：
1. `200` 请求成功
2. `400` 参数或业务校验失败
3. `404` 资源不存在
4. `500` 服务内部错误
5. `503` 应用状态未初始化

## 3. 数据模型

### 3.1 CreateTaskRequest
用于创建下载任务（`POST /api/tasks`）。

```json
{
  "url": "https://www.youtube.com/watch?v=example",
  "output_dir": "D:/Downloads",
  "download_target": "video",
  "format_id": "bestvideo+bestaudio",
  "resolution": "1080",
  "resolution_mode": "prefer",
  "audio_format": "mp3",
  "subtitle_mode": "none",
  "subtitle_langs": "zh-Hans,en.*",
  "write_info_json": false,
  "write_description": false,
  "write_thumbnail": false,
  "embed_thumbnail": false,
  "cookie_mode": "none",
  "cookie_value": ""
}
```

字段说明：
1. `url`：视频链接，必填
2. `output_dir`：输出目录，必填
3. `download_target`：下载目标，`video | audio | thumbnail`
4. `format_id`：格式 ID，可选（仅 `video` 目标生效）
5. `resolution`：分辨率，可选（仅 `video` 目标生效）
6. `resolution_mode`：分辨率策略，`prefer | limit`（仅 `video` 目标生效）
7. `audio_format`：音频格式，可选（仅 `audio` 目标生效）
8. `subtitle_mode`：字幕模式，`none | manual | auto | all`
9. `subtitle_langs`：字幕语言表达式，可选
10. `write_info_json`：是否输出 `.info.json`
11. `write_description`：是否输出 `.description`
12. `write_thumbnail`：是否写出封面图片
13. `embed_thumbnail`：是否嵌入封面到媒体文件
14. `cookie_mode`：`none | file | browser`
15. `cookie_value`：cookie 来源值（文件路径或浏览器名），可选

### 3.2 DownloadTask
任务完整结构（列表、创建返回、WebSocket 推送都会用到）。

```json
{
  "id": "uuid",
  "url": "https://www.youtube.com/watch?v=example",
  "output_dir": "D:/Downloads",
  "download_target": "video",
  "format_id": "bestvideo+bestaudio",
  "resolution": "1080",
  "resolution_mode": "prefer",
  "audio_format": null,
  "subtitle_mode": "none",
  "subtitle_langs": null,
  "write_info_json": false,
  "write_description": false,
  "write_thumbnail": false,
  "embed_thumbnail": false,
  "cookie_mode": "none",
  "cookie_value": null,
  "status": "pending",
  "progress": 0,
  "speed": null,
  "eta": null,
  "log": "",
  "error": null,
  "result_path": null,
  "created_at": "2026-05-11T10:20:30.123456",
  "updated_at": "2026-05-11T10:20:30.123456"
}
```

`status` 可选值：
1. `pending`
2. `running`
3. `completed`
4. `failed`
5. `canceled`

### 3.3 TaskActionResponse
用于取消/删除/清空等动作接口返回。

```json
{
  "ok": true,
  "message": "Task canceled"
}
```

## 4. HTTP 接口

### 4.1 健康说明
#### GET `/`
说明：根路径说明接口，用于确认后端是否运行。

响应示例：

```json
{
  "message": "yt-dlp desktop backend is running",
  "hint": "Use /api/* endpoints from desktop frontend"
}
```

---

### 4.2 任务接口（`/api/tasks`）

#### GET `/api/tasks`
说明：获取当前内存任务列表（按创建时间倒序）。

响应：`DownloadTask[]`

#### POST `/api/tasks`
说明：创建任务并加入队列。

请求体：`CreateTaskRequest`  
响应：`DownloadTask`

失败示例（400）：

```json
{
  "detail": "Dependencies are still installing"
}
```

```json
{
  "detail": "Dependencies are not installed"
}
```

#### POST `/api/tasks/{task_id}/cancel`
说明：取消任务（仅未结束任务可取消）。

路径参数：
1. `task_id`：任务 ID

响应：`TaskActionResponse`

失败示例（404）：

```json
{
  "detail": "Task not found or not cancelable"
}
```

#### DELETE `/api/tasks/{task_id}`
说明：删除队列任务（`running` 状态不可直接删除，建议先取消）。

路径参数：
1. `task_id`：任务 ID

响应：`TaskActionResponse`

失败示例（404）：

```json
{
  "detail": "Task not found or running"
}
```

---

### 4.3 历史接口（`/api/history`）

#### GET `/api/history?limit=200`
说明：读取 SQLite 历史记录。

查询参数：
1. `limit`：返回上限，范围 `1-1000`，默认 `200`

响应：历史记录数组（结构与 `DownloadTask` 兼容）

#### DELETE `/api/history/{task_id}`
说明：删除单条历史记录。

路径参数：
1. `task_id`：任务 ID

响应：`TaskActionResponse`

失败示例（404）：

```json
{
  "detail": "History item not found"
}
```

#### DELETE `/api/history`
说明：清空全部历史记录。

响应：`TaskActionResponse`

响应示例：

```json
{
  "ok": true,
  "message": "History cleared: 12"
}
```

---

### 4.4 Cookie 接口（`/api/cookies`）

#### POST `/api/cookies/import`
说明：校验用户选择的 `cookies.txt` 文件路径是否有效。

请求体：

```json
{
  "path": "D:/path/to/cookies.txt"
}
```

响应示例：

```json
{
  "ok": "true",
  "path": "D:/path/to/cookies.txt"
}
```

失败示例（400）：

```json
{
  "detail": "Cookie 文件不存在或格式非法"
}
```

---

### 4.5 系统接口（`/api/system`）

#### GET `/api/system/dependencies`
说明：查看本地依赖文件存在性（`yt-dlp` / `ffmpeg`）。

响应示例：

```json
{
  "yt_dlp": {
    "path": "D:/.../yt-dlp.exe",
    "exists": true,
    "installing": false
  },
  "ffmpeg": {
    "path": "D:/.../ffmpeg.exe",
    "exists": true,
    "installing": false
  }
}
```

字段补充：
1. `installing`：是否正在下载安装该依赖（启动自动安装或手动更新时为 `true`）

#### GET `/api/system/yt-dlp/update-status`
说明：检查 `yt-dlp` 更新状态。

响应示例：

```json
{
  "installed_version": "2026.05.01",
  "latest_version": "2026.05.10",
  "has_update": true,
  "binary_path": "D:/.../yt-dlp.exe"
}
```

#### POST `/api/system/yt-dlp/update`
说明：一键下载/更新 `yt-dlp`。

响应示例：

```json
{
  "installed_version": "2026.05.01",
  "latest_version": "2026.05.10",
  "has_update": true,
  "binary_path": "D:/.../yt-dlp.exe",
  "updated_to": "2026.05.10",
  "asset": "yt-dlp.exe",
  "path": "D:/.../yt-dlp.exe",
  "updated": true
}
```

失败示例（500）：

```json
{
  "detail": "yt-dlp update failed: 具体错误"
}
```

#### GET `/api/system/ffmpeg/update-status`
说明：检查 `ffmpeg` 更新状态。

响应示例：

```json
{
  "installed_version": "ffmpeg version ...",

  "local_release_id": 123000000,

  "binary_path": "D:/.../ffmpeg.exe"
}
```

#### POST `/api/system/ffmpeg/update`
说明：一键下载/更新 `ffmpeg`。

响应示例：

```json
{
  "installed_version": "ffmpeg version ...",
  "latest_release_id": 123456789,
  "latest_tag_name": "latest",
  "latest_published_at": "2026-05-10T00:00:00Z",
```

## 5. WebSocket 接口

### 5.1 任务与终端事件流
#### WS `/ws/tasks`
说明：统一事件流，前端通过一个连接接收任务状态变化和终端日志输出。

事件结构：

```json
{
  "type": "事件类型",
  "data": {}
}
```

事件类型：
1. `task_update`：任务状态更新，`data` 为 `DownloadTask`
2. `terminal_output`：终端输出事件，`data` 结构如下

`terminal_output` 示例：

```json
{
  "type": "terminal_output",
  "data": {
    "task_id": "system-yt-dlp",
    "stream": "stdout",
    "text": "latest yt-dlp release: 2026.05.10, assets=24"
  }
}
```

`stream` 常见值：
1. `command`：命令或请求动作
2. `stdout`：标准输出内容
3. `status`：状态提示

## 6. 调用示例（cURL）

### 6.1 创建任务
```bash
curl -X POST "http://127.0.0.1:8000/api/tasks" ^
  -H "Content-Type: application/json" ^
  -d "{\"url\":\"https://www.youtube.com/watch?v=example\",\"output_dir\":\"D:/Downloads\",\"cookie_mode\":\"none\"}"
```

### 6.2 取消任务
```bash
curl -X POST "http://127.0.0.1:8000/api/tasks/{task_id}/cancel"
```

### 6.3 删除队列任务
```bash
curl -X DELETE "http://127.0.0.1:8000/api/tasks/{task_id}"
```

### 6.4 删除单条历史
```bash
curl -X DELETE "http://127.0.0.1:8000/api/history/{task_id}"
```

### 6.5 清空历史
```bash
curl -X DELETE "http://127.0.0.1:8000/api/history"
```

### 6.6 检查并更新 yt-dlp
```bash
curl "http://127.0.0.1:8000/api/system/yt-dlp/update-status"
curl -X POST "http://127.0.0.1:8000/api/system/yt-dlp/update"
```

### 6.7 检查并更新 ffmpeg
```bash
curl "http://127.0.0.1:8000/api/system/ffmpeg/update-status"
curl -X POST "http://127.0.0.1:8000/api/system/ffmpeg/update"
```

## 7. 版本与维护建议
1. 每次新增/修改路由时同步更新本文档。
2. 若前端提示 `Method Not Allowed / Not Found`，优先检查是否运行了旧版后端进程。
3. 可用 `GET /openapi.json` 作为运行时接口真值对照。
