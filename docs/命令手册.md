# YouTubeDownloader 命令手册（小白版）

这份文档只讲“你现在就能直接复制运行”的命令。
当前版本以 Windows 平台为主要发布目标。

## 1. 第一次准备（只需要做一次）

在项目根目录执行：

```bash
npm install --no-audit --no-fund
python -m pip install -r backend/requirements.txt
```

## 2. 开发启动（推荐）

```bash
npm run dev
```

说明：
- 这条命令会同时启动前端开发服务 + Electron 桌面端。
- 这是最不容易白屏的开发方式。

## 3. 只启动桌面端（不启动前端开发服务）

```bash
npm run build:frontend
npm run dev:electron
```

说明：
- 第一步先把前端打包到 `frontend/dist`。
- 第二步启动 Electron，Electron 会自动加载打包后的页面。

## 4. 单独启动后端（调试 API 用）

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload --app-dir backend
```

说明：
- 浏览器访问 `http://127.0.0.1:8000/` 会返回运行提示信息（不是前端页面）。
- 真正接口在 `/api/*` 下。

## 5. 构建代码（不打包安装包）

```bash
npm run build
```

它会依次执行：
- `npm run build:frontend`
- `npm run build:electron`

## 6. 打包安装包

建议先执行一次图标生成（确保 `.exe` 使用最新图标）：

```bash
npm run generate:icons
```

### 当前平台打包

```bash
npm run dist
```

### Windows 打包（推荐）

```bash
npm run dist:win
```

### 仅产出未封装目录（排查打包问题时很有用）

```bash
npm run pack
```

## 7. 常见问题

### 7.1 桌面端白屏

按顺序排查：
1. 先执行 `npm run build:frontend`
2. 再执行 `npm run dev:electron`
3. 看 Electron 开发者工具 Console 是否仍有 `ERR_FILE_NOT_FOUND`

### 7.2 打开 8000 端口不是前端页面

这是正常的：
- `8000` 是后端 API 服务，不是前端页面。
- 根路径通常会返回后端运行提示信息。
- 桌面前端由 Electron 加载。

### 7.3 更新 yt-dlp / ffmpeg

在桌面端“设置”页点击：
- `检查更新`
- `下载/更新`

程序会自动把可执行文件放到 `resources/bin/windows/`。

补充说明（当前版本行为）：
- 应用启动时会自动检查依赖文件是否存在，缺失时会自动后台下载安装。
- 自动下载安装期间，“下载”页的“添加任务”会被禁用，并提示“依赖未安装/未安装完成”。
- 你可以在“日志”页查看依赖下载过程输出。

### 7.4 图标没有更新（仍显示默认 Electron 图标）

按顺序排查：
1. 执行 `npm run generate:icons`
2. 重新执行 `npm run dist:win`
3. 卸载旧安装包后再安装新包（Windows 图标缓存可能导致旧图标残留）
